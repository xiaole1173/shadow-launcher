## 优先级建议

### 🔴 P0（必须立即修复，强校验缺失导致无提示崩溃）
1. **java.exe 存在性检查** — 最常见的启动失败原因
2. **核心 Jar 文件存在性** — 版本未安装或损坏直接炸
3. **natives 库完整性** — 缺失闪退无任何提示
4. **内存是否超过系统可用** — 可能卡死电脑
5. **进程启动后 5 秒内存活检测** — 已崩溃但 overlay 显示"完成"

### 🟡 P1（应该尽快修复）
6. 离线用户名格式校验
7. 正版账户 token 过期检测
8. Java 32/64 位架构 + 内存限制
9. 目录读写权限
10. 磁盘空间预警

### 🟢 P2（后续迭代）
11. 模组兼容性检测
12. 残留进程检测
13. 资源文件完整性（弱校验）
14. JVM 参数语法校验
