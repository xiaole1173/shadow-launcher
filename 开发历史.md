# 2026-06-15（第一天 — 项目启动 Day 0）

## 16:26–18:00 — QML 架构搭建

### 问题

项目采用 **PySide6 6.11.1 + QQmlApplicationEngine + FramelessWindowHint** 作为初始架构。搭建完成后发现：所有 `pageContainer` 内 `MouseArea` 的 `onClicked` / `onPressed` 事件全部不触发，整个页面交互完全失效。

### 方案探索

- **方案 A（已否决）**：改用 Qt Widgets 体系，放弃 QML 声明式 UI。评估后认为工作量过大，且丧失了 QML 动画和声明式布局的优势。
- **方案 B（采纳）**：深入排查 QML 事件传递机制。

### 根因分析

经逐层排查发现：所有 `Loader` 元素均设置为 `anchors.fill: parent`，多层 Loader 互相重叠覆盖。QML 中 `z` 属性不控制事件传递层级——即使后方的 Loader 不可见，只要其 `active` 为 true 且 `visible` 为 true，就会拦截鼠标事件。

### 最终修复

为所有 Loader 添加条件绑定：

```qml
Loader {
    active: condition
    visible: active
    source: "..."
}
```

非活动 Loader 直接销毁（`active: false`），彻底释放事件传递路径。

### 结果

✅ 全部按钮可点击，事件系统恢复正常。

## 22:36–23:49 — 安装系统端到端修复

### 问题

整个安装流程阻塞——点击安装后无任何进展反馈。

### 根因分析

`version_manifest`（Mojang 版本清单）未预加载，安装线程启动时无法获取版本元数据。

### 解决方案

| 组件 | 说明 |
|------|------|
| `install_cli.py` | CLI 安装工具，通过 stdout 管道输出进度 |
| `testing_click.py` | 自动化安装测试脚本 |

### Bug 修复

**问题**：emoji 字符（如 Minecraft 版本名中的特殊字符）导致 `subprocess` GBK 编码崩溃。

**修复**：
1. stdout 输出前做 sanitize 处理，过滤不可编码字符
2. `json.dumps` 配合 `flush` + `os.fsync` 确保数据可靠落盘

### 首次成功安装

| 版本 | 文件数 | 大小 |
|------|--------|------|
| Minecraft 1.20.4 | 3,875 | 24.4 MB |

### 结果

✅ 安装系统端到端打通，首个完整版本安装成功。

## 核心踩坑经验

1. **QML Loader 事件吞噬**：多层 Loader 的 `anchors.fill: parent` 会互相覆盖，需 `active: condition` + `visible: active` 控制生命周期。
2. **GBK 编码崩溃**：subprocess stdout 含 emoji 时 GBK 崩溃，需 sanitize + UTF-8 管道。
3. **Java 架构检测**：PE 头检测（0s）远优于 `-XshowSettings`（2–10s）。
4. **UTF-8 编码损坏**：子 Agent `edit` 工具可能损坏非 ASCII 文件，复杂修改应用 Python 安全脚本。
5. **PyInstaller vs Nuitka**：中文路径环境 Nuitka 不兼容，PyInstaller 稳定可用。
6. **版本隔离**：每个 MC 版本独立目录，存档/MOD/资源包全部跟随隔离。
